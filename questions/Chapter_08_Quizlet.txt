D413 – Review Questions – Chap 8 Answers 
 
1. Summarize the history of UNIX operating systems. UNIX's history began in 
1969 at Bell Labs. It evolved from a small project initially used for text processing 
of patent documents. It laid the groundwork for many modern operating systems, 
including Linux and macOS. 
2. What is the native communication protocol used for UNIX? Most UNIX-
based systems natively use TCP/IP as their network communication protocol. 
3. Why is UNIX/Linux a favorite among programmers and programming 
enthusiasts? UNIX/Linux is favored by programmers and enthusiasts for several 
reasons: 
o Free and Open Source: Its source code is accessible, allowing for 
customization, modification, and distribution without license fees. 
o Customization and Control: It offers unparalleled flexibility, enabling 
developers to modify the system, install only necessary components, and 
create streamlined environments. 
o Stability and Reliability: Linux systems are known for their robustness, 
running smoothly for long periods without crashes or reboots, making 
them ideal for servers and development. 
o Large Community Support: A strong community fosters collaboration, 
providing extensive resources and assistance. 
o Compatibility with Development Tools: It's highly compatible with a 
wide range of automation tools and frameworks used in DevOps (e.g., 
Docker, Kubernetes, Ansible). 
o Efficiency and Low Resource Requirements: It can run on various 
hardware with relatively low memory and disk space requirements. 
4. What is a daemon? A daemon is a computer program that runs in the 
background, performing various tasks without direct user interaction. They are 
common in Unix and Unix-like operating systems like Linux, acting as helpful 
assistants for system services. 
5. Summarize the advantages and disadvantages of using the Linux operating 
system. 
Advantages: 
o Open Source: Free to use, modify, and distribute. 
o Secure & Reliable: Known for stability, strong security features, and 
infrequent crashes. 
o Cost-Effective: No licensing expenses. 
o Highly Customizable: Allows users to tailor the system to their specific 
needs. 
o Large Community Support: Extensive online resources and help 
available. 
o Ideal for Programming: Provides a robust environment with excellent 
tool compatibility. 
o Lightweight: Can run on older hardware with fewer resources. 
Disadvantages: 
o Steep Learning Curve: Can be more challenging for new users, 
especially with command-line interfaces. 
o Hardware Compatibility Issues: Some hardware devices may have 
limited or no open-source drivers. 
o Software Interoperability: Not all software is readily available for Linux, 
as many developers focus on Windows and macOS. 
o Less Gaming Support: Generally has poorer gaming support compared 
to Windows. 
6. Describe the purpose of a GNU General Public License. The GNU General 
Public License is a family of copyleft software licenses that guarantee end-users 
the freedom to run, study, share, and modify software. A key aspect of copyleft is 
that any derivative work distributed must also be under the same or equivalent 
license terms, ensuring that the software remains free.  
7. What is the purpose of LILO and GRUB? Both LILO (Linux Loader) and GRUB 
(GRand Unified Bootloader) are bootloaders for Linux. Their purpose is to 
manage the boot process by loading the Linux kernel into memory and initiating 
the boot sequence. They also allow users to choose between multiple operating 
systems (in dual-boot setups) or different kernel versions installed on the system. 
GRUB is more modern and feature-rich than LILO, supporting a wider range of 
operating systems, file systems, and dynamic reconfiguration. 
8. What is the function of a shell? A shell is a command-line interpreter that acts 
as an interface between the user and the operating system (like UNIX/Linux). Its 
primary function is to collect user input (commands) and execute them, allowing 
users to interact efficiently and directly with the operating system. 
9. List three common shells used with UNIX/Linux. Three common shells are: 
o Bash (Bourne-Again SHell) 
o Zsh (Z Shell) 
o Csh (C Shell) or Tcsh (TENEX C Shell, an enhanced version of Csh) 
o Ksh (Korn Shell) 
10. List and describe Linux-specific file systems. Some common Linux-specific 
file systems include: 
o Ext4 (Fourth Extended File System): The most widely used and default 
file system for many Linux distributions, it is a journaling file system and 
successor to Ext3. 
o Btrfs (B-Tree File System): A modern file system offering features like 
snapshots, checksums, and integrated volume management. 
o XFS: A 64-bit journaling file system, often used for large file systems and 
high-performance computing, and a default for some distributions. 
o Ext2 (Second Extended File System): An older non-journaling file 
system, still used in some cases like flash drives or SSDs where 
journaling overhead might not be desired. 
o Ext3 (Third Extended File System): A journaling file system that adds 
journaling capabilities to Ext2. 
o JFS (Journaled File System): Developed by IBM, it's a journaling file 
system ported to Linux. 
o F2FS (Flash-Friendly File System): Designed specifically for NAND flash 
memory-based storage devices (like SSDs). 
11. What is a journaling file system? A journaling file system is a file system that 
maintains a special log, or "journal," of changes that are yet to be committed to 
the main part of the file system. This journal allows for quick recovery from 
system crashes or power failures by replaying or rolling back incomplete 
operations, preventing data corruption and long delays during system boot-up. 
12. Using the DOS file system as a comparison, describe the UNIX/Linux file 
system.  
UNIX/Linux File System: 
o Hierarchical Structure: Organized in a single inverted tree-like structure, 
starting from the root directory (/). All files and directories are located 
under this root. 
o Case-Sensitive: Filenames and directory names distinguish between 
uppercase and lowercase (e.g., File.txt is different from file.txt). 
o Forward Slashes: Uses forward slashes (/) as directory separators (e.g., 
/home/user/document.txt). 
o Multi-user and Multitasking: Designed to support multiple users and run 
multiple processes concurrently. 
o Permissions: Robust file permissions (read, write, execute) for users, 
groups, and others. 
DOS File System (FAT): 
o Flat/Per-Drive Structure: Each drive (C:, D:, etc.) has its own separate 
root directory. 
o Case-Insensitive: Filenames and directory names do not distinguish 
between uppercase and lowercase. 
o Backslashes: Uses backslashes (\) as directory separators (e.g., 
C:\USERS\DOCUMENT.TXT). 
o Single-user and Single-tasking: Originally designed for single-user, 
single-task environments. 
o Limited Permissions: Offers basic file attributes (read-only, hidden, 
system, archive) but lacks the granular user/group permissions of 
UNIX/Linux. 
Similarities: 
o Both are command-line operable. 
o Both support basic file management operations. 
13. All files and all directories on a UNIX/Linux system are located under the 
_____ directory. All files and all directories on a UNIX/Linux system are located 
under the root directory (indicated by /). 
14. What is a mount point? A mount point is a directory or file in the existing file 
system hierarchy where another file system, directory, or file is attached and 
made accessible. When you "mount" a file system, you are essentially linking its 
root to a specific directory within the main file system tree. 
15. What are the three Linux file and directory permissions and how are they 
indicated? The three Linux file and directory permissions are: 
o Read (r): Allows viewing the contents of a file or listing the contents of a 
directory. 
o Write (w): Allows modifying the contents of a file or creating/deleting files 
within a directory. 
o Execute (x): Allows running a file as a program or entering/traversing a 
directory. 
They are typically indicated by the letters 'r', 'w', and 'x' or by numerical values: 
o r = 4 
o w = 2 
o x = 1 
These permissions are assigned to three categories: 
o User (u): The owner of the file. 
o Group (g): Members of the group associated with the file. 
o Others (o): Everyone else. 
For example, rwx (7) means read, write, and execute permission, while r-x (5) 
means read and execute permission. 
16. What is the difference between FTP and TFTP? 
o FTP (File Transfer Protocol): 
▪ 
Uses TCP (Transmission Control Protocol) for reliable, connection-
oriented transfers. 
▪ 
Establishes two connections: one for control (commands and 
responses, on port 21) and another for data transfer (on a separate, 
ephemeral port). 
▪ 
Provides user authentication (username and password). 
▪ 
Supports a wider range of features, including directory listings, 
resume transfers, and both binary and text modes. 
▪ 
More robust and suitable for transferring larger files over WANs, but 
by default, data is sent in plain text (FTPS uses SSL for encryption). 
o TFTP (Trivial File Transfer Protocol): 
▪ 
Uses UDP (User Datagram Protocol) for unreliable, connectionless 
transfers. 
▪ 
Does not provide user authentication or many other features of 
FTP. 
▪ 
Simpler and designed for transferring smaller files, often used for 
booting diskless workstations or transferring configuration files in 
local networks. 
▪ 
Lacks built-in encryption mechanisms. 
17. What is Telnet used for? Telnet is a network protocol that allows users to 
connect to and communicate with remote computers over a TCP/IP network. It 
essentially provides a virtual terminal, enabling access to the command-line 
interface of a remote system as if the user were physically present. However, it's 
considered insecure for sensitive data as it transmits data in plain text. 
18. Describe an X Windows system. The X Window System (also known as X11) is 
a network-transparent windowing system that provides a GUI environment for 
UNIX/Linux and other Unix-like operating systems. It operates on a client-server 
model: 
o X Server: Runs on the user's local machine (display), managing input 
(keyboard, mouse) and output (screen display). 
o X Client: The application program that runs on a remote machine (e.g., a 
server) and sends display commands to the X server. 
This architecture allows an application running on one computer to open windows 
and display its output on another computer's screen across a network. 
19. Summarize interoperability between UNIX/Linux systems and Windows 
systems. Interoperability between UNIX/Linux and Windows systems is 
generally achieved through several mechanisms: 
o Networking Protocols: Both systems widely support standard network 
protocols like TCP/IP, enabling communication for file sharing (SMB/CIFS 
for Windows shares, NFS for Linux shares), printing, and remote access 
(SSH from Linux to Windows with appropriate tools, or Telnet/SSH from 
Windows to Linux). 
o File Sharing: Common protocols like SMB/CIFS (Samba on Linux) allow 
Linux systems to access Windows shares, and NFS allows Windows to 
access Linux shares (with NFS client installed). 
o Remote Access: SSH (Secure Shell) is the primary method for secure 
command-line access to Linux from Windows, and various tools exist for 
graphical remote access (like VNC or RDP to a Windows desktop from 
Linux). 
o Cross-Platform Applications: Many applications are designed to run on 
both platforms, or open-source alternatives are available. 
o Virtualization: Running one OS as a virtual machine within another (e.g., 
Linux in VirtualBox on Windows, or Windows in a VM on Linux). 
o WSL (Windows Subsystem for Linux): A feature in Windows 10/11 that 
allows users to run a Linux environment directly on Windows, providing 
seamless integration. 
o Naming Conventions and Unicode: Challenges can arise with file and 
directory naming conventions (case sensitivity, allowed characters, path 
separators) and Unicode handling, requiring careful configuration for full 
compatibility, especially in backup/restore scenarios. 
20. Discuss macOS Server. macOS Server was a series of Unix-like server 
operating systems developed by Apple Inc., based on macOS. It aimed to 
provide server functionality and system administration tools for managing macOS 
and iOS devices, along with network services like mail, file sharing (AFP, SMB), 
LDAP, and DNS, as well as server applications (web, database). 
Historically, it was a standalone operating system (e.g., Mac OS X Server 1.0, 
released in 1999). However, starting with OS X Lion, Apple discontinued selling a 
standalone server OS and instead offered an add-on "Server app" through the Mac 
App Store. This Server app gradually lost many of the features from the full server 
OS, eventually focusing primarily on user/group management, Xsan, and mobile 
device management. 
macOS Server (the Server app) was officially discontinued by Apple in 2022, with 
later versions of macOS dropping support for it. This marked the end of Apple's 
dedicated server operating system product line.
