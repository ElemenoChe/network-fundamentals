D413 – Review Questions – Chap 16 Answers 
 
1. Explain the concept of elasticity as it relates to cloud computing. Elasticity 
in cloud computing refers to the ability of a system to scale computing 
resources up or down rapidly and automatically to meet changing demand. 
This means that resources can be provisioned or de-provisioned as needed, 
often in real-time, without human intervention. It ensures that users always have 
enough resources to handle their workloads, but also that they don't pay for 
resources they aren't using. 
2. What are the five characteristics of cloud computing? According to the NIST 
(National Institute of Standards and Technology), the five essential 
characteristics of cloud computing are: 
o On-demand self-service: Users can provision computing capabilities, 
such as server time and network storage, automatically without requiring 
human interaction with each service provider. 
o Broad network access: Capabilities are available over the network and 
accessed through standard mechanisms that promote use by 
heterogeneous thin or thick client platforms (e.g., mobile phones, laptops, 
and workstations). 
o Resource pooling: The provider's computing resources are pooled to 
serve multiple consumers using a multi-tenant model, with different 
physical and virtual resources dynamically assigned and reassigned 
according to consumer demand. 
o Rapid elasticity: Capabilities can be elastically provisioned and released, 
in some cases automatically, to scale rapidly outward and inward 
commensurate with demand. To the consumer, the capabilities available 
for provisioning often appear to be unlimited and can be appropriated in 
any quantity at any time. 
o Measured service: Cloud systems automatically control and optimize 
resource use by leveraging a metering capability at some level of 
abstraction appropriate to the type of service (e.g., storage, processing, 
bandwidth, and active user accounts). Resource usage can be monitored, 
controlled, and reported, providing transparency for both the provider and 
consumer. 
3. Define multitenancy. Multitenancy is an architecture in which a single instance 
of a software application or a single physical infrastructure serves multiple 
customers (tenants). Each tenant's data is isolated and remains invisible to 
other tenants, even though they share the same underlying resources. This 
allows cloud providers to achieve economies of scale and optimize resource 
utilization. 
4. Describe the four common technologies used in virtualization.  
o Hardware Virtualization (or Server Virtualization): This is the most 
common form, where a hypervisor creates virtual machines (VMs) that 
emulate a complete computer system, allowing multiple operating systems 
to run concurrently on a single physical server. 
o Network Virtualization: This involves abstracting network resources (like 
switches, routers, firewalls, and load balancers) from the underlying 
hardware. It allows for the creation of virtual networks that can be 
managed independently of the physical network infrastructure. 
o Storage Virtualization: This combines multiple physical storage devices 
into a single, logical storage unit. This allows for easier management, 
increased utilization, and improved data protection. 
o Application Virtualization: This allows applications to run in an isolated 
environment, separate from the underlying operating system. This can 
include packaging applications to run on a variety of devices without 
installation or providing remote access to applications.  
5. What are the four deployment models most focused on by the NIST? The 
NIST defines four cloud deployment models: 
o Private Cloud: The cloud infrastructure is provisioned for exclusive use by 
a single organization comprising multiple consumers (e.g., business units). 
It may be owned, managed, and operated by the organization, a third 
party, or some combination of them, and it may exist on or off premises. 
o Community Cloud: The cloud infrastructure is provisioned for exclusive 
use by a specific community of consumers from organizations that have 
shared concerns (e.g., mission, security requirements, policy, and 
compliance considerations). It may be owned, managed, and operated by 
one or more of the organizations in the community, a third party, or some 
combination of them, and it may exist on or off premises. 
o Public Cloud: The cloud infrastructure is provisioned for open use by the 
general public. It may be owned, managed, and operated by a business, 
academic, or government organization, or some combination of them. It 
exists on the premises of the cloud provider. 
o Hybrid Cloud: The cloud infrastructure is a composition of two or more 
distinct cloud infrastructures (private, community, or public) that remain 
unique entities, but are bound together by standardized or proprietary 
technology that enables data and application portability (e.g., cloud 
bursting for load balancing between clouds). 
6. What is the difference between polycloud and multicloud? 
o Multicloud: Refers to the use of two or more cloud services from 
different cloud providers (e.g., using AWS for one application and Azure 
for another) or different services from the same provider (e.g. using Azure 
for IaaS and AWS for PaaS). The key is diversification of cloud 
environments. 
o Polycloud: While sometimes used interchangeably with multicloud, 
polycloud often implies a more strategic and intentional adoption of 
multiple cloud services, each chosen for its specific strengths or 
best-of-breed capabilities for different workloads or applications. It's less 
about simply distributing workloads and more about optimizing each 
workload for the most suitable cloud environment, leading to a potentially 
more complex but also more optimized cloud landscape. 
7. Explain the three service models defined by the NIST. The NIST defines three 
cloud service models: 
o Software as a Service (SaaS): The capability provided to the consumer is 
to use the provider's applications running on a cloud infrastructure. The 
applications are accessible from various client devices through a thin 
client interface such as a web browser (e.g., web-based email) or a 
program interface. The consumer does not manage or control the 
underlying cloud infrastructure including network, servers, operating 
systems, storage, or even individual application capabilities, with the 
possible exception of limited user-specific application configuration 
settings. (Examples: Gmail, Salesforce, Dropbox) 
o Platform as a Service (PaaS): The capability provided to the consumer is 
to deploy onto the cloud infrastructure consumer-created or acquired 
applications created using programming languages, libraries, services, 
and tools supported by the provider. The consumer does not manage or 
control the underlying cloud infrastructure including network, servers, 
operating systems, or storage, but has control over the deployed 
applications and possibly configuration settings for the application-hosting 
environment. (Examples: Google App Engine, Heroku, AWS Elastic 
Beanstalk) 
o Infrastructure as a Service (IaaS): The capability provided to the 
consumer is to provision processing, storage, networks, and other 
fundamental computing resources where the consumer is able to deploy 
and run arbitrary software, which can include operating systems and 
applications. The consumer does not manage or control the underlying 
cloud infrastructure but has control over operating systems, storage, 
deployed applications, and possibly limited control of select networking 
components (e.g., host firewalls). (Examples: AWS EC2, Azure Virtual 
Machines, Google Compute Engine) 
8. What are the two types of Desktop as a Service? The two main types of 
Desktop as a Service (DaaS) are: 
o Persistent DaaS: Users are provided with a dedicated, persistent virtual 
desktop that retains all their settings, data, and installed applications 
between sessions. This offers a personalized and familiar desktop 
experience, much like a traditional physical desktop. 
o Non-Persistent DaaS: Users are provided with a generic, "clean" virtual 
desktop each time they log in. Any changes made to the desktop are 
typically discarded upon logout. This type is often used for task-based 
workers or environments where users don't need to save data directly to 
the desktop. User profiles and data are usually stored separately. 
9. What are some common technical challenges of cloud computing? 
Common technical challenges of cloud computing include: 
o Security and Compliance: Data breaches, unauthorized access, 
compliance with regulations (GDPR, HIPAA), and shared security 
responsibilities can be complex. 
o Vendor Lock-in: Difficulty in migrating data and applications from one 
cloud provider to another due to proprietary technologies, APIs, and 
services. 
o Performance and Latency: Network latency can affect application 
performance, especially for geographically dispersed users or latency-
sensitive applications. 
o Data Gravity: The tendency for large datasets to "pull" applications and 
services towards them, making it challenging to move data between cloud 
environments. 
o Integration Complexity: Integrating on-premises systems with cloud 
services, or integrating different cloud services, can be challenging. 
o Cost Management: While cloud computing can be cost-effective, 
managing and optimizing cloud spending (finops) can be complex due to 
dynamic pricing models and usage-based billing. 
o Data Residency and Sovereignty: Legal and regulatory requirements 
regarding where data must be stored and processed. 
o Network Connectivity and Bandwidth: Reliable and high-bandwidth 
internet connectivity is crucial for cloud services. 
o Monitoring and Management: Gaining comprehensive visibility and 
control over resources scattered across multiple cloud environments. 
10. What does ISO 27018 focus on? ISO/IEC 27018:2019 (Information technology 
— Security techniques — Code of practice for protection of personally identifiable 
information (PII) in public clouds acting as PII processors) focuses on privacy 
protection in cloud computing environments, specifically for personally 
identifiable information (PII). It provides guidelines for cloud service providers 
on how to protect PII when it is processed in the public cloud, acting as PII 
processors. It builds upon ISO/IEC 27002 and adds specific guidance for cloud 
environments, emphasizing consent, transparency, and data subject rights.

massive scale and comprehensive offerings: 
o Amazon Web Services (AWS)
